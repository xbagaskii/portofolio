About Me

I am a cybersecurity enthusiast specializing in web application security, vulnerability research, and practical penetration testing. My journey into security began through self-directed learning, driven by curiosity about how systems work and how they can be secured against real-world threats. Without a formal academic background in computer science, I built my expertise through hands-on experimentation, independent research, and continuous practice in real-world environments. I focus primarily on identifying and analyzing vulnerabilities in web applications, including issues such as Cross-Site Scripting (XSS), insecure API implementations, authentication flaws, and misconfigurations. My approach to learning cybersecurity is highly practical. I regularly work with security tools and methodologies used by professionals in the industry, including traffic interception, API analysis, vulnerability scanning, and manual testing techniques. I believe that real understanding in security comes from breaking down systems, analyzing their behavior, and learning from each discovery. In addition to vulnerability research, I also build small security tools and automation scripts to assist with reconnaissance, testing workflows, and data analysis. These projects reflect my interest in combining programming with offensive security techniques. Currently, my main focus is: Web Application Security Testing Bug Bounty Methodology API Security Analysis Security Automation & Tool Development Vulnerability Research I actively continue improving my skills through continuous research, security labs, bug bounty platforms, and hands-on experimentation. My goal is to grow as a professional security researcher and contribute to making web technologies safer.
